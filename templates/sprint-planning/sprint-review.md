## Example: Sprint 23 Review

**Sprint Goal:** Implement user profile editing  
**Goal Achieved:** ✅ Yes

### Completed Work

**Story 1: Edit Basic Profile Info**
- **Demo by:** Sarah (Frontend Dev)
- **What we built:** Users can now edit name, email, phone
- **Key features:**
  - Inline editing
  - Real-time validation
  - Auto-save every 30 seconds
- **Business value:** Reduces support tickets for profile updates

**Demo Flow:**
1. Log in as test user
2. Navigate to profile
3. Click "Edit" button
4. Change name and email
5. Show auto-save indicator
6. Refresh page to confirm changes persisted

**Story 2: Profile Picture Upload**
- **Demo by:** Mike (Full-stack Dev)
- **What we built:** Users can upload and crop profile pictures
- **Key features:**
  - Drag-and-drop upload
  - Image cropping tool
  - Automatic resizing
- **Business value:** Increases profile completion rate

**Story 3: Privacy Settings**
- **Demo by:** Alex (Backend Dev)
- **What we built:** Users control who sees their profile
- **Key features:**
  - Public/Private toggle
  - Granular field-level privacy
  - Preview as another user
- **Business value:** Addresses privacy concerns from user research

### Metrics
- Committed: 25 points, Completed: 25 points (100%)
- Velocity: 25 points (on track)
- Bugs found: 2 minor (fixed before demo)
- Code coverage: 87% (up from 83%)

### Feedback Received

**VP Product:**
- "Love the auto-save feature!"
- Request: Add undo button for accidental changes
- Action: Added to backlog as P2

**Customer Success Manager:**
- "This will reduce 30% of our support tickets"
- Request: Add help tooltips for privacy settings
- Action: Quick win, adding to next sprint

**Beta User (Sarah from Acme Corp):**
- "Image cropping is intuitive"
- Issue: Confused about public vs. private
- Action: Improve copy, add tooltip

### Next Sprint
**Goal:** Implement notification preferences
**Top Stories:**
- Email notification settings (8 pts)
- Push notification settings (5 pts)
- Notification digest (5 pts)

## Sprint Review Best Practices

### DO:
✅ Prepare demos in advance  
✅ Use real data (or realistic test data)  
✅ Focus on working software, not slides  
✅ Encourage questions and feedback  
✅ Keep it interactive and engaging  
✅ Celebrate team successes  
✅ Be honest about what didn't work  
✅ Demo in production-like environment  

### DON'T:
❌ Read through bullet points  
❌ Demo non-working features  
❌ Make excuses for incomplete work  
❌ Go over time  
❌ Skip this meeting (it's not optional!)  
❌ Turn it into a status meeting  
❌ Demo only to the PM (invite stakeholders!)  
❌ Wing it without preparation  

---

## Demo Preparation Checklist

**1 Day Before:**
- [ ] Test demo environment
- [ ] Create test data/accounts
- [ ] Prepare demo script
- [ ] Notify stakeholders
- [ ] Set up screen sharing
- [ ] Prepare metrics/slides

**Day Of:**
- [ ] Verify demo environment still works
- [ ] Have backup plan if demo fails
- [ ] Start on time
- [ ] Record session (if remote)
